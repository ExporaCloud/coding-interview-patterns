# Contributing

This is an open guide to the core coding interview patterns. Contributions that make it clearer, more accurate, or more complete are welcome.

## What fits
- Corrections to triggers, complexity, or problem mappings.
- A clearer explanation of a mechanic.
- A well-known classic problem missing from a pattern's list (include its LeetCode number).
- A pattern distinction that trips people up.

## What does not fit
- Full solution code dumps. This repo teaches recognition, not memorization.
- Language-specific implementations. Templates stay as language-agnostic pseudocode.
- Self-promotion or unrelated links.

## How to submit
1. Fork the repo and create a branch.
2. Keep the style: concise, trigger-first, one idea per section.
3. Open a pull request describing what you changed and why.

Thanks for helping developers understand algorithms instead of memorizing them.
